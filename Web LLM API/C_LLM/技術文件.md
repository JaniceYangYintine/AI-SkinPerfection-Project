## 專案概述

### 專案目標
開發一個基於 Gemini AI 的健康建議系統,透過問卷收集使用者的生活習慣和健康狀況,提供個人化的飲食、保養和生活建議。

### 核心功能
- 🌐 **前端網頁**: 問卷表單介面,支援快速測試功能
- 🤖 **後端 API**: Flask + Gemini AI 整合
- 📊 **日誌記錄**: 完整記錄每次問卷和 AI 回應
- 🔧 **測試工具**: 快速填充和隨機填充功能

### 技術棧
| 層級 | 技術 | 版本 |
|------|------|------|
| 前端 | HTML5, CSS3, JavaScript | - |
| 後端 | Python, Flask | 3.1.0 |
| AI | Google Gemini | 2.5 Flash |
| 資料存儲 | JSON | - |

## 技術架構

### 系統架構圖
```
┌─────────────┐      HTTP      ┌──────────────────┐      HTTP      ┌─────────────┐
│   瀏覽器    │ ←────────────→ │  前端伺服器       │                │             │
│ (使用者)    │   Port 8080    │ frontend_server  │                 │             │
└─────────────┘                └──────────────────┘                 │             │
                                                                    │             │
                                        │                           │   Gemini    │
                                        │ HTTP/JSON                 │     AI      │
                                        ↓                           │             │
                               ┌──────────────────┐      API       │             │
                               │   後端 API       │ ←────────────→ │             │
                               │   Flask + AI     │                │             │
                               │   Port 5000      │                └─────────────┘
                               └──────────────────┘
                                        │
                                        ↓
                               ┌──────────────────┐
                               │   logs/          │
                               │   history.json   │
                               └──────────────────┘
```

### 資料流程
1. 使用者在瀏覽器填寫問卷
2. 前端發送 POST 請求到 `/api/health-advice`
3. 後端 API 接收問卷數據
4. 生成 prompt 並發送給 Gemini AI
5. 接收 AI 回應
6. 記錄到 `history.json`
7. 返回結果給前端
8. 前端顯示健康建議

## 開發過程與問題解決

### 問題 1: CORS 跨域問題

**問題描述**:
前端網頁無法直接呼叫後端 API,出現 CORS (Cross-Origin Resource Sharing) 錯誤。

**原因分析**:
- 前端運行在 `localhost:8080`
- 後端運行在 `localhost:5000`
- 不同 port 被視為不同來源,瀏覽器預設阻擋跨域請求

**解決方案**:
```python
# 方案 1: 使用 flask-cors (推薦)
from flask_cors import CORS
CORS(app)

# 方案 2: 手動設定 CORS headers
@app.after_request
def after_request(response):
    response.headers.add('Access-Control-Allow-Origin', '*')
    response.headers.add('Access-Control-Allow-Headers', 'Content-Type')
    response.headers.add('Access-Control-Allow-Methods', 'GET,POST')
    return response
```

**學習重點**:
- 理解同源政策 (Same-Origin Policy)
- 學習 CORS 的運作機制
- 了解 preflight request

---

### 問題 2: Port 佔用錯誤

**問題描述**:
執行 `start.bat` 時出現錯誤:
```
OSError: [WinError 10048] 一次只能用一個通訊端位址
```

**原因分析**:
- Port 5000 或 8080 已被其他程序佔用
- 通常是之前的伺服器沒有正確關閉

**解決方案**:

**方案 1: 自動清理腳本** (`cleanup_ports.bat`)
```batch
# 檢查並終止佔用 port 的程序
for /f "tokens=5" %%a in ('netstat -ano ^| findstr ":5000"') do (
    taskkill /F /PID %%a
)
```

**方案 2: 改進 start.bat**
```batch
# 啟動前自動檢查並清理 port
echo [檢查] 正在檢查 Port 5000...
for /f "tokens=5" %%a in ('netstat -ano ^| findstr ":5000"') do (
    taskkill /F /PID %%a >nul 2>&1
)
```

**學習重點**:
- Windows 網路診斷命令 (`netstat`)
- 程序管理 (`taskkill`)
- 批次腳本編寫

---

### 問題 3: Windows 控制台編碼問題

**問題描述**:
Python 程式中的 emoji 和中文在 Windows 控制台顯示為亂碼或導致程式崩潰。

**錯誤訊息**:
```
UnicodeEncodeError: 'cp950' codec can't encode character '\u26a0'
```

**原因分析**:
- Windows 預設使用 CP950 (Big5) 編碼
- Emoji 和部分 Unicode 字元無法在 CP950 中表示

**解決方案**:

**方案 1: 設定 UTF-8 編碼**
```python
import sys
if sys.platform == 'win32':
    sys.stdout.reconfigure(encoding='utf-8')
```

**方案 2: 移除 emoji,使用純文字**
```python
# 原本
print("✅ 系統啟動成功")

# 改為
print("[OK] 系統啟動成功")
```

**方案 3: 批次檔設定 UTF-8**
```batch
chcp 65001 >nul
```

**學習重點**:
- 字元編碼的重要性
- Windows 和 Unix 系統的編碼差異
- Python 的編碼處理

### 問題 4: JSON 日誌可讀性差

**問題描述**:
`history.json` 中的 `prompt` 和 `ai_response` 欄位內容很長,全部擠在一行,需要水平捲動很遠才能看完。

**原因分析**:
- JSON 的 `indent` 參數只會縮排物件和陣列
- 長字串不會自動換行

**解決方案**:
將長文本轉換為行陣列:
```python
# 將長文本按行分割
if 'prompt' in log and log['prompt']:
    log['prompt'] = log['prompt'].split('\n')
if 'ai_response' in log and log['ai_response']:
    log['ai_response'] = log['ai_response'].split('\n')
```

**效果對比**:
```json
// 舊格式 (難讀)
{"prompt": "第一行\n第二行\n第三行"}

// 新格式 (易讀)
{"prompt": ["第一行", "第二行", "第三行"]}
```

**學習重點**:
- JSON 格式化技巧
- 資料可視化的重要性
- 使用者體驗優化

## 功能實作細節

### 1. 日誌記錄系統

**設計目標**:
- 記錄每次問卷提交和 AI 回應
- 包含完整的 prompt 和模型資訊
- 易於查看和分析

**實作方式**:
```python
def save_log(user_data, ai_response, status="success", 
             error_message=None, model_name=None, prompt=None):
    log_entry = {
        "id": str(uuid.uuid4()),
        "timestamp": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
        "model_name": model_name,
        "user_input": user_data,
        "prompt": prompt,
        "ai_response": ai_response,
        "status": status
    }
    
    # 格式化長文本
    formatted_logs = []
    for log in logs:
        formatted_log = log.copy()
        if 'prompt' in formatted_log and formatted_log['prompt']:
            formatted_log['prompt'] = formatted_log['prompt'].split('\n')
        if 'ai_response' in formatted_log and formatted_log['ai_response']:
            formatted_log['ai_response'] = formatted_log['ai_response'].split('\n')
        formatted_logs.append(formatted_log)
    
    # 寫入 JSON
    with open(LOGS_FILE, 'w', encoding='utf-8') as f:
        json.dump(formatted_logs, f, ensure_ascii=False, indent=2)
```

**關鍵技術**:
- UUID 生成唯一 ID
- 時間戳記錄
- JSON 格式化
- 錯誤處理

### 3. Prompt Engineering

**設計思路**:
- 明確的角色設定
- 結構化的輸出格式
- 字數限制確保簡潔

**Prompt 結構**:
```python
prompt = f"""
【角色設定】
你是一位溫柔體貼的「健康護理師」💕

【使用者數據】
- 年齡: {user_data.get('age')}
- 膚質: {user_data.get('skin_type')}
...

【任務目標】
請根據以上數據,為這位使用者生成專屬建議。

【輸出格式要求】
1. 字數限制: 每個建議 15 字以內
2. 排版要求: 每個建議之間空一行
3. 保養步驟: 給出清楚的順序編排

###生活改善建議
建議內容

###營養補充建議
建議內容
...
"""
```

**優化技巧**:
- 使用明確的分隔符 (`###`)
- 提供範例格式
- 限制字數避免冗長

## 改進建議

### 短期改進 (1-2 週)

#### 1. 資料庫整合
**現況**: 使用 JSON 檔案儲存
**問題**: 
- 大量資料時效能差
- 不支援複雜查詢
- 並發寫入可能出錯

**建議**:
```python
# 使用 SQLite
import sqlite3

# 建立資料表
CREATE TABLE logs (
    id TEXT PRIMARY KEY,
    timestamp DATETIME,
    model_name TEXT,
    user_input JSON,
    prompt TEXT,
    ai_response TEXT,
    status TEXT
)
```

**優點**:
- 更好的效能
- 支援 SQL 查詢
- 資料完整性保證

---

#### 2. 錯誤處理增強
**現況**: 基本的 try-except
**問題**: 錯誤訊息不夠詳細

**建議**:
```python
import logging

# 設定日誌
logging.basicConfig(
    filename='logs/error.log',
    level=logging.ERROR,
    format='%(asctime)s - %(levelname)s - %(message)s'
)

# 使用
try:
    result = model.generate_content(prompt)
except Exception as e:
    logging.error(f"AI 生成失敗: {e}", exc_info=True)
    return {"error": "系統錯誤,請稍後再試"}
```

```python
    default_limits=["10 per minute"]

@app.route('/api/health-advice', methods=['POST'])
@limiter.limit("5 per minute")

    # ...
```
