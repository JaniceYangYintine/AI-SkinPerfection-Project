## 🔍 如何檢查套件依賴

### 方法 1: 檢查已安裝的套件

```bash
# 列出所有已安裝的套件
pip list

# 列出已安裝套件並顯示版本
pip freeze

# 只顯示專案需要的套件 (推薦)
pip freeze > installed.txt
```

**輸出範例**:
```
Flask==3.1.0
flask-cors==5.0.0
google-generativeai==0.8.6
python-dotenv==1.2.1
...
```

---

### 方法 2: 檢查特定套件是否安裝

```bash
# 檢查單一套件
pip show Flask

# 檢查多個套件
pip show Flask flask-cors python-dotenv google-generativeai
```

**輸出範例**:
```
Name: Flask
Version: 3.1.0
Summary: A simple framework for building complex web applications.
Home-page: https://palletsprojects.com/p/flask
Author: Armin Ronacher
License: BSD-3-Clause
Location: c:\...\site-packages
Requires: Werkzeug, Jinja2, itsdangerous, click
Required-by: flask-cors
```

---

### 方法 3: 驗證 requirements.txt 的套件

```bash
# 檢查 requirements.txt 中的套件是否都已安裝
pip check

# 安裝 requirements.txt 中的所有套件
pip install -r requirements.txt

# 升級 requirements.txt 中的所有套件
pip install --upgrade -r requirements.txt
```

---

### 方法 4: 找出缺少的套件

```bash
# 嘗試安裝,如果已安裝會顯示 "Requirement already satisfied"
pip install -r requirements.txt

# 只顯示需要安裝的套件
pip install -r requirements.txt --dry-run
```

---

### 方法 5: 自動生成 requirements.txt

```bash
# 方法 A: 使用 pip freeze (包含所有依賴)
pip freeze > requirements.txt

# 方法 B: 使用 pipreqs (只包含專案實際使用的套件,推薦)
pip install pipreqs
pipreqs . --force
```

**pipreqs 的優點**:
- 只列出專案實際使用的套件
- 不包含開發環境的其他套件
- 更乾淨、更精確

## ⚠️ 常見問題

### Q1: 出現 "ModuleNotFoundError"
**原因**: 缺少該套件

**解決方法**:
```bash
# 安裝缺少的套件
pip install 套件名稱

# 或重新安裝所有套件
pip install -r requirements.txt
```

---

### Q2: 版本衝突
**錯誤訊息**: 
```
ERROR: pip's dependency resolver does not currently take into account...
```

**解決方法**:
```bash
# 方法 1: 移除版本限制
# 將 Flask==3.1.0 改為 Flask

# 方法 2: 使用相容版本
# 將 Flask==3.1.0 改為 Flask>=3.0.0
```

---

### Q3: 套件安裝失敗
**解決方法**:
```bash
# 升級 pip
python -m pip install --upgrade pip

# 清除快取
pip cache purge

# 重新安裝
pip install -r requirements.txt
```

## 🎯 最佳實踐

### 1. 使用虛擬環境
```bash
# 建立虛擬環境
python -m venv .venv

# 啟動虛擬環境 (Windows)
.venv\Scripts\activate

# 啟動虛擬環境 (macOS/Linux)
source .venv/bin/activate

# 在虛擬環境中安裝套件
pip install -r requirements.txt
```

**優點**:
- 隔離專案依賴
- 避免版本衝突
- 方便部署

---

### 2. 固定版本號
```txt
# 推薦: 固定版本
Flask==3.1.0

# 不推薦: 不固定版本
Flask

# 可接受: 最低版本
Flask>=3.0.0
```

**原因**: 固定版本確保在不同環境中行為一致

---

### 3. 定期更新套件
```bash
# 檢查過期套件
pip list --outdated

# 更新特定套件
pip install --upgrade Flask

# 更新所有套件 (謹慎使用)
pip install --upgrade -r requirements.txt
```
